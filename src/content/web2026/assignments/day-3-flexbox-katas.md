In class we did the first four exercises of [**web2026-space-oddity**](https://github.com/open-making/web2026-space-oddity). Tonight you finish the last three.

**Due at the start of Day 4's class. Time-box: 75 minutes total.** Work in your fork of the repo; each exercise lives in its own folder with its own README.

### 05: Profile list (write the HTML)

This time the CSS is finished and you write the HTML to fit it. Read `style.css` as a spec: it tells you which classes and nesting it expects. Build a short list of people with your own names and roles. Producing markup that fits an existing stylesheet is most of what working on a real codebase is.

### 06: Card (build the whole thing)

No starter code, just a screenshot (`goal.png`). Mark the boxes first, like the Day 3 warm-up, then write the HTML and style it. It's a profile card: a header row with two sides, and a row of three stats below. The idea to get is nesting, a row whose children are themselves columns. Match the layout, not the exact pixels.

### 07: Drills (three small ones)

Three patterns you'll use constantly: a toolbar with a button pushed to the right, a search row where the input grows, and a box centered in a band. Froggy levels 1 to 13 cover all of them if you want a warm-up.

### Stretch (optional): 08 and 09

Two extra exercises use flexbox to lay out a whole page editorially: 08 is a readable article column, and 09 is a magazine front page built entirely with flexbox. Try them if you finish early or want more. They are a taste of the layout work we do all next week.

### Submitting

Commit and push your fork with GitHub Desktop, then drop the link to your fork in the submission thread along with your dev note. (Optional: deploy it to Netlify too, if you'd like a live link.)

Stuck past your time-box? Stuck-note it. Flexbox Froggy levels 1 to 13 cover everything these exercises need, so revisiting the game *is* a legitimate solving strategy.
